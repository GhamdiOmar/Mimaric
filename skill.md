# SaaS UI/UX Excellence Skill

**Description:** Use this skill when the user asks to create, refactor, or design UI components, dashboards, pricing pages, or landing pages. This skill ensures the interface follows professional design standards, avoids "vibe-coded" AI mistakes, and implements core UX concepts like visual hierarchy and interaction feedback.

## Instructions

### 1. Avoid AI "Vibe-Code" Mistakes
*   **Icons:** Never use emojis for professional interfaces. Replace them with professional icon libraries like Lucide or Phosphor icons. [5]
*   **Color Selection:** Do not use randomly selected bright colors. Start with one primary brand color and derive backgrounds and text colors from it. [5, 6] Use semantic colors (blue for trust, red for danger, yellow for warning, green for success) for purpose rather than decoration. [6]
*   **Layout Repetition:** Ensure KPIs and data points are not repeated unnecessarily across the same page. [7, 8] Avoid repeating identical dashboard cards. [7]
*   **Pricing Tables:** Keep plans simple, ideally limiting options to 3-4 plans. [9] Clearly highlight the discount the user gets for annual billing and show what the next tier includes that the current one does not. [8]

### 2. Core UI/UX Concepts
*   **Visual Hierarchy:** Use size, position, and color to draw attention to the most important elements. Most important items should be large, bold, and positioned near the top. [10, 11]
*   **Spacing & Whitespace:** Implement a 4-point grid system where all spacing is a multiple of 4 or 8 to create consistency. [12, 13] Use ample whitespace between sections to let the design "breathe." [12]
*   **Typography:** Stick to one high-quality sans-serif font for the entire design. [13] For large headers, tighten the letter spacing by -2% to -3% and set the line height to 110-120% to achieve a "pro" look. [14]
*   **Interaction Feedback:** Every interactive element must provide a response. Buttons require four states: default, hovered, active/pressed, and disabled. [15] Use micro-interactions, such as a "Copied!" chip sliding up, to confirm user actions immediately. [16]
*   **Depth & Shadows:** In light mode, use subtle shadows with low opacity and high blur. [17] In dark mode, do not use shadows; instead, create depth by making cards slightly lighter than the background. [17]
*   **Simplifying Layouts:** Reduce visual noise by collapsing secondary buttons into triple-dot menus and tucking secondary links into popovers. [7, 18] Use modals for simple data entry fields rather than sparse full-page layouts. [18]
